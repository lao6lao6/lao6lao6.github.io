# 爬取当当网畅销榜top500
## 1. 项目背景与目标

### 1.1 背景
在信息爆炸的时代，获取图书市场的畅销信息对于出版社、作者和读者都具有重要价值。当当网作为国内知名的图书电商平台，其图书畅销榜数据具有较高的参考价值。

### 1.2 目标
- 开发一个自动化爬虫，爬取当当网图书畅销榜的完整数据
- 实现对1-25页数据的批量爬取，共500条图书信息
- 将爬取的数据以结构化格式保存，便于后续分析和使用
- 确保爬虫的稳定性和可靠性，应对网站反爬机制

## 2. 技术栈选择

| 技术/库 | 版本 | 用途 |
|--------|------|------|
| Python | 3.x | 主要开发语言 |
| requests | 2.25.1+ | 发起HTTP请求 |
| re | 内置库 | 正则表达式解析HTML |
| json | 内置库 | 数据序列化与存储 |
| time | 内置库 | 控制爬取频率，避免被封 |

## 3. 爬虫架构设计

### 3.1 整体架构
```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  URL生成    │────>│ HTTP请求    │────>│ HTML解析    │────>│ 数据存储    │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

### 3.2 核心模块
1. **URL生成模块**：根据页码规则生成完整的URL
2. **HTTP请求模块**：模拟浏览器发送请求，处理异常情况
3. **HTML解析模块**：使用正则表达式提取图书信息
4. **数据存储模块**：将解析后的数据以JSON格式写入文件

## 4. 核心功能实现

### 4.1 HTTP请求处理
```python
def request_dandan(url):
    try:
        response = requests.get(url, headers=headers, timeout=15)
        # 确保使用正确的编码
        response.encoding = 'gb2312'
        if response.status_code == 200:
            # 保存页面内容到文件，方便调试

                with open('bestsellers_page.html', 'w', encoding='UTF-8') as f:
                    f.write(response.text)
            return response.text
        else:
            print(f'请求失败，状态码: {response.status_code}')
            return None
    except requests.RequestException as e:
        print(f'请求异常: {e}')
        return None
```

### 4.2 HTML解析
```python
def parse_result(html):
    if not html:
        return
    
    try:
        # 使用更准确的正则表达式模式，匹配畅销榜页面的HTML结构
        pattern = re.compile(
            r'<li>.*?<div class="list_num.*?">(+).*?</div>.*?<img src="(.*?)".*?title="(.*?)".*?<span class="tuijian">(.*?)</span>.*?<div class="publisher_info">(.*?)</div>.*?<span class="price_n">(.*?)</span>', re.S)
        items = re.findall(pattern, html)
        print(f'解析到 {len(items)} 个项目')

        for item in items:
            # 清理数据，去除多余的HTML标签
            recommend = re.sub(r'[^\d%]', '', item[3])
            author = re.sub(r'<.*?>', '', item[4])
            price = re.sub(r'[^\d.]', '', item[5])
            
            # 确保数据编码正确
            yield {
                'range': item[0],
                'image': item[1],
                'title': item[2],
                'recommend': recommend + '%推荐',
                'author': author.strip(),
                'times': '0次',  # 畅销榜页面没有评分次数
                'price': '¥' + price
            }
    except Exception as e:
        print(f'解析异常: {e}')
        return
```

## 5. 代码优化过程

### 5.1 初始问题
- 正则表达式过于严格，无法匹配动态HTML结构
- 编码处理不当，导致中文乱码
- 缺少异常处理，爬虫稳定性差
- 没有反爬机制应对策略

### 5.2 优化措施
1. **正则表达式优化**：使用更灵活的模式匹配，适应HTML结构变化
2. **编码处理**：明确设置response.encoding = 'gb2312'，确保中文正确显示
3. **异常处理**：添加try-except块，捕获请求和解析过程中的异常
4. **反爬策略**：添加请求头模拟浏览器，设置合理的爬取延迟
5. **调试支持**：保存页面内容到文件，方便分析HTML结构
